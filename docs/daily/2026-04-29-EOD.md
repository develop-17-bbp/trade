### 3.2 Self-awareness + learning channels (commits 18, 20)

**`prediction_accuracy.py`** — reads warm_store outcomes, buckets recent predictions by `direction × conviction_tier`, computes per-bucket win-rate, surfaces:
- `calibration_label`: `well_calibrated` / `over_confident` / `under_confident`
- `top_miss_reasons`: aggregated from `self_critique.miss_reasons`
- One-line `PREDICTION_ACCURACY:` summary auto-injected into `tick_state` every tick

So the brain reads — every tick, in its prompt — *how accurate its own past predictions have been, broken down by direction and conviction tier*.

**`brain_health.py`** — meta-discipline tracker. 60-second TTL cache so the multi-asset executor loop pays cost once per minute, not per asset:
- `tool_variety_score = distinct_tools_used / 99 available` (low → "defaulting to a small subset")
- `avg_thesis_quality_score`: 0-1 heuristic — does the brain's thesis cite factor_synthesis bias? DSR? similar setups? regime? sniper? pattern_score? ml_meta? macro? dominance? cvd? whale? halving?
- `under_trading` / `over_trading` / `focus_drift` flags
- Auto-injected as `BRAIN_HEALTH:` summary line; `LOW_TOOL_VARIETY` / `WEAK_THESIS` flag set when thresholds tripped

Brain now sees its own discipline as a numeric input; operator sees it on the dashboard.

**`eod_review.py`** — daily learning distillation:
- Reads today's closed trades from warm_store
- Surfaces 3 best + 3 worst, detects regime / tier / miss-reason patterns, suggests adjustments for tomorrow
- Writes Markdown to `memory/eod/<YYYY-MM-DD>.md`
- Compact `YESTERDAY_REVIEW:` line auto-injected into `tick_state` for the first ticks of next day so brain inherits yesterday's lessons across the UTC roll
- Date-keyed cache so per-tick disk reads aren't pure waste

Closes the gap between the in-conversation `<critique>` traces (which drift out of the 5-trace context window after a day) and durable per-day learning.

### 3.3 ML / strategy infrastructure (commits 5, 6, 7, 8)

**Backtest rigor (commit 5, `09ef4c8`):**
- **Deflated Sharpe Ratio** (Bailey-López de Prado 2014) — discounts Sharpe by trial count to account for selection bias from running 20+ hypothesis sweeps
- **Probability of Backtest Overfitting (PBO)** (Bailey et al. 2017) — combinatorially symmetric cross-validation, returns probability that the in-sample winner is actually below-median out-of-sample
- **Purged Walk-Forward** (López de Prado 2017) — purge embargo around test fold to defeat leakage from overlapping label horizons
- **Adaptive slippage model** — fits slippage as `f(spread, bar_volume, our_size, hour_of_day)` from realized fills

These don't change live trading directly; they harden the genetic-strategy promotion path so promoted strategies are less likely to be paper-tigers.

**Catalyst preemption + continuous-brain daemon + decision graph (commit 6, `27a19e2`):**
- `continuous_brain.py` — non-tick-blocking daemon that maintains a rolling micro-prediction stream the executor can read mid-tick without waiting for full ReAct
- `catalyst_listener.py` — pre-empts the tick when news/macro catalyst arrives that flips factor_synthesis bias
- `decision_graph.py` — embedding-based similar-setup retrieval; brain can ask "find decisions where my state was similar"

**4 frontier ML modules (commit 7, `10c74b7`):**

## 5. Anti-overfit / anti-noise / anti-hallucination posture

Every new feature carries explicit guards (operator's recurring concern: "make sure LLM's are not getting overfitted, noise, hallucinations, etc."):

- **Bounded outputs.** Every numeric signal is in `[-1, +1]`, `[0, 1]`, or a small fixed set of regime labels. No unbounded floats reach the brain.
- **Time-decay + cache TTLs.** External APIs (DXY/VIX/BTC.D) cache 5 min — prevents rate-limit hammering AND prevents the bot from chasing single-tick noise.
- **Sample-size warnings.** `prediction_accuracy.calibration_label` returns `low_sample` until 20+ trades; `brain_health.sample_warning` until 20+ ticks observed; `eod_review.skipped_reason` triggers below 3 trades/day. Brain reads these, doesn't act on under-sampled signals.
- **N-bar persistence.** CVD divergence requires 5-bar agreement before firing — single-bar noise filtered.
- **Safe-DSL whitelist.** LLM-generated alphas can only reference whitelisted feature symbols + arithmetic; no `eval` / `exec` / unbounded arithmetic.
- **Champion-gate as ONLY behavior-change path.** New strategies / new ML adapters / new alphas all evaluate against incumbent; ≥ 2% improvement on direction-agreement AND no metric regression > 5% required.
- **Shadow-mode default.** Three integrations from commit 2 default OFF; promotion is operator-gated.
- **Snapshot persistence.** Factor snapshot at trade submission means miss attribution is *deterministic* — if calibration drifts toward `over_confident`, the operator can pull the exact factor regimes that drove the over-confident calls.
