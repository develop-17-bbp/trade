## 2. Day's Objectives (in priority order)

### Objective A — Auditability / Reasoning-Trace Artifact (HIGH PRIORITY)

**Why:** Current research on agentic AI in finance flags **opacity** as the dominant blocker for regulator/auditor/investor trust in multi-agent trading systems. ACT logs all the underlying data (warm_store decisions, brain_memory traces, agent votes, tool dispatches) but does not render any single per-decision artifact that an external reviewer can read to reconstruct *why* a trade was placed. This is also a precondition the operator needs for authority submission of trading-system output.

**Deliverable:** A new structured `reasoning_trace` field attached to every `TradePlan`, persisted to `warm_store.decisions`, exposing:
- ordered list of tool dispatches with name + key inputs + digest of output
- agent-vote summary (direction + confidence per voting agent)
- gate verdicts in order (authority → conviction → cost → readiness → safe-entries → protections)
- final synthesis paragraph
- input-stream snapshot (PORTFOLIO/GOAL/RATCHET/COST lines from tick_state at decision time)

**Surface:** New brain tool `query_reasoning_trace(plan_id)` returning the artifact; new API endpoint `/api/v1/reasoning_trace/<plan_id>` for external review.

**Acceptance criteria:**
1. Every new `TradePlan` produced by `agentic_bridge.compile_agentic_plan` carries a `reasoning_trace` dict, persisted on submit.
2. Brain tool returns the trace given a `plan_id`.
3. API endpoint returns the trace as JSON with stable schema.
4. `tests/test_reasoning_trace.py` validates schema + persistence + retrieval.
5. `pytest tests/ -k "reasoning_trace or agentic_bridge or unified_brain_tools"` passes.

**Estimate:** 4-5 hours.

### Objective B — Natural-Language Post-Trade Reflection (MEDIUM PRIORITY)

**Why:** Current in-context learning channels (`recent_critiques`, `analyst_traces`) inject **structured data** into the next-tick prompt. The FinMem and TradingAgents research papers report measurable performance lift from injecting a **one-paragraph natural-language reflection** ("what I tried, what happened, what I'd do differently") generated by the same analyst LLM after each trade close. The natural-language form parses more cleanly into the next analyst pass than a dataclass dump.

**Deliverable:** Extension to `trade_verifier.verify_outcome` that, in addition to the existing structured `SelfCritique`, generates a **single 100-150 word reflection paragraph** scoped to the closed trade. Stored under `decisions.reflection_paragraph` in warm_store. Auto-injected into `recent_critiques` seed context (already wired via `agentic_bridge.load_recent_critiques`).

**Surface:** No new tool or endpoint — the paragraph flows through the existing critique injection path. New field appears in `query_trade_verifier_state` results.

**Acceptance criteria:**
1. `trade_verifier.verify_outcome` returns a `SelfCritique` whose `lessons` field contains a coherent 100-150 word natural-language reflection.
2. The paragraph is auto-rendered into the next-tick analyst seed (verifiable via `format_for_brain` smoke test with a mock recent close).
3. `tests/test_trade_verifier.py` adds a case asserting paragraph presence and length bound.
4. Targeted pytest passes.

**Estimate:** 2-3 hours.

### Objective C — Cross-Asset Thesis Sharing (LOW PRIORITY, FILLER)

**Why:** Current per-asset reasoning sees the *count* of opens on the other asset (`open_positions_other_assets`) and the cointegration signal, but not the *current thesis* of the other asset's most recent decision. This produces occasional contradictory positioning when correlation suggests inverse direction. Trivial to fix — the data is in `analyst_traces`, just not joined cross-asset.

**Deliverable:** When the brain reasons about BTC, inject ETH's latest analyst-trace direction + tier + thesis as a single `OTHER_ASSET_THESIS:` line in the evidence document. Vice versa for ETH.

**Surface:** Update to `agentic_bridge.compile_agentic_plan` and `tick_state.format_for_brain`.

**Acceptance criteria:**
1. BTC tick context includes ETH's most-recent thesis (and vice versa) when present.
2. Smoke test demonstrates the line appears in `format_for_brain` output.

**Estimate:** 30-45 minutes.
